
In this lab, we will build our first neural


network model with TensorFlow and deploy the


model on the embedded board with STM Cube AI.


We will implement a letter recognition model


which takes accelerometer data from the board


and predicts the letter based on that data.


For example, if you draw the letter O


with the board, then the model will


predict which letter you drew.


You will also train your first neural


network model with TensorFlow and


deploy the inference with STM Cube AI.


You will implement the activity


recognition application using a multilayer


perceptron and compare the model accuracy


with and without feature extraction.


Finally, you will evaluate the


performance on an STM32 microcontroller.


In this lab,


we're going to use TensorFlow to train


multilayer perceptron models and use STM

You can see that the training and

This means that the 200 epochs


are enough to train the model.


If we train the model for too many epochs,


the validation accuracy may


drop due to overfitting.


For the training,


we used the default learning


rate for the model,


but how would the model change


with different learning rates if


the learning rate is too high?


The model is more likely


to overshoot the minima,


meaning that the model cannot


converge to the minima.


On the other hand,


if the learning rate is too low,


the model reaches the minima too slowly,


requiring more training time.


Let's first try a high learning rate.


Here the learning rate of


the optimizer is set as 1,000.


Execute the code block.


This graph shows the training and


validation loss values against epochs.


You can see that the loss


values fluctuate a lot,


so the model has difficulty


in reaching the minima.


Let's try a lower learning


rate such as 0.0001.


Execute the code block.


The graph shows that the training


and validation loss values


decrease much more slowly,


so it is important to use a proper


learning rate for training.
