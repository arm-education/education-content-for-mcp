Let's deploy the model on the board


using STM Cube AI and STM Cube IDE.


First, we'll see how to validate


the model on the board.


STM Cube AI is an extension


software of STM Cube MX.


It is not installed by default,


so we need to install it before we begin.


Open STM Cube MX.


Click access to board selector.


Find your board,


and Click start project.


To configure the project,


go to Project Manager,


write the project name and


select the project location


where the project will be saved.


This figure shows the


configuration of pinouts.


Actually, we don't need to use all of them,


so we are going to clear


them. On the pinout menu,


click clear pinouts.


Now we are going to install Cube AI


and enable it for your project.


On the Software Packs menu,


click select components.


Click the install button.


Expand the menu and enable Cube AI for


device application, choose validation.


You can see that Cube AI is now enabled.


On this cube AI menu, click add network.


Hey, you can load your model and


the validation input and output.


When you click analyse, it generates


a detailed report on the model.


You can check the number of parameters,


the size of the weights and the


amount of memory to be used.


When you click show graph,


it displays a visualization of your model.


When you click validate on desktop,


it validates the model with the provided


input and output on your desktop.


To validate the model on the target,


we need to first generate the code and


install that code on the target.


Click generate code and it


automatically generates the code for


validating the model on the board.


Open STM Cube IDE.


On this menu, choose STM 32


project from an existing STM


32CubeMX configuration file.


Then navigate to the project


folder and select the IOC file.


Install the code with run as.


Now it is installing the Cube


AI application to the board.


Go back to the STM Cube MX


and press validate on target.


Now the model is validated,


let's implement an


application with the model.


Open the list of software packages.


Change the device application


with application.


Click generate code again to update


the project. Open STM Cube IDE.


Since we regenerated the code,


we need to delete the existing


project and import the project again.


Open X-CUBE-AI app.


App_x-cube-ai.c.


This is the autogenerated


code by STM Cube MX.


Between user code begin and user code end


you can write code for your application.


There are two main functions that


you should fill. The first function


is acquire and process data.


This function is for getting data


from sensors and processing the data.


For example, extracting features.


The second function is post process


for post processing the output.


For the demo, we already implemented


a sample application so you can test


the application with your board.


Open any terminal application to


see the output from the board.


We will use Tera Term in this lab.


Set up the serial connection.


The speed is 115,200.


Import the application, click


the file tab and click import.


Select existing project into Workspace.


Go to Misc and select MCU


activity recognition.


Open X-CUBE-AI. App_x-cube-ai.c.


We already filled in the functions for


data acquisition from the accelerometer


sensor and for feature extraction.


Install the code with runas.


Now you can see the output on the terminal.


Press the blue button on the board, draw a


letter, then press the blue button again.
