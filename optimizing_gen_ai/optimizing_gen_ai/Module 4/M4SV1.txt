metrics.

So when you input a prompt that's a
string,

it first needs to be tokenized and then
your large language model and the

transformer needs to imbue all the
context between the different tokens and

this is called the prompt processing rate
and it corresponds to the time to first

token.

It takes a long time to process the input
prompt.

Your first token output might take a
while.

The equation here is a use to derive that
equation from llama.cpp's metrics.

So we divide the number of prompt tokens
by the top prompt processing rate and

then add that to the reciprocal of one
over the token generation rate,

which gives us the time to first token.

Secondly,
the total inference latency is the same

number of prompt tokens divided by the
prompt processing rate,

but then added to the number of output
tokens, divided by token generation rate.

So this is all related to the
responsiveness of the system.

Even if you have a really fast token
generation rate,

if the preprocessing time takes a while
